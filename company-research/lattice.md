# Lattice Research

## Conceptual definition

Anduril describes Lattice as an AI-powered operating system that turns thousands of data streams into a real-time, 3D command-and-control environment.

Source captured July 10, 2026: https://job-boards.greenhouse.io/andurilindustries/jobs/5130552007

## What Josh should understand

At a conceptual level, Lattice must integrate information from multiple sensors, platforms, networks, operators, and software systems into a coherent operational picture.

That creates mission-analytics problems such as:

- Identity and entity resolution
- Sensor fusion and conflicting observations
- Time synchronization
- Track continuity
- Version and configuration context
- Low-latency data delivery
- Data quality under degraded connectivity
- Access control and classification boundaries
- Historical replay and forensic analysis
- Trusted metrics for leaders and operators

## Interview-safe explanation

> My understanding is that Lattice creates a shared operational picture by integrating many data streams across sensors and autonomous systems. From a mission-analytics perspective, the important challenge is preserving enough lineage, timing, configuration, and quality context that users can trust both the real-time picture and the historical analysis built from it.

## Practice questions

1. Why is a shared entity model important when many systems describe the same object?
2. How would you distinguish a real system anomaly from a bad measurement?
3. What metadata would you preserve for historical replay?
4. How would intermittent connectivity affect a real-time operational picture?
5. Why are software and configuration versions essential analytical dimensions?
6. How would you communicate confidence or uncertainty to an operator?

## Avoid

- Claiming detailed knowledge of proprietary Lattice architecture
- Speculating about classified capabilities
- Reducing Lattice to "a dashboard"
- Speaking as if machine learning removes the need for data quality and validation
