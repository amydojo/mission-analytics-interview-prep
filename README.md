# Mission Analytics Interview Prep

Private preparation system for Joshua Gross's Anduril Industries phone screen for the Mission Analytics Engineer role.

## Core campaign message

> Systems and software engineer with defense, RF, Python automation, SQL-driven analytics, production debugging, and graduate statistics experience.

Josh is not pitching himself as a generic data scientist trying to enter defense. He is presenting a coherent progression from physical and networked systems into mission analytics.

## Phone-screen win condition

The recruiter should leave believing:

1. Josh has credible defense and RF systems experience.
2. He has used Python and SQL to diagnose real production behavior.
3. He understands reliability, validation, and root-cause analysis.
4. He communicates clearly across technical and nontechnical audiences.
5. His gaps in specific analytics-stack tools are learnable rather than foundational.

## Start here

1. Read [`role-analysis/requirements-to-evidence.md`](role-analysis/requirements-to-evidence.md).
2. Complete every `[JOSH: ...]` prompt in the story files.
3. Rehearse the answer files aloud rather than memorizing them word-for-word.
4. Run the mock interview in [`mock-interviews/recruiter-screen.md`](mock-interviews/recruiter-screen.md).
5. Use [`day-of/one-page-cheat-sheet.md`](day-of/one-page-cheat-sheet.md) on interview day.

## Safety

Do not add classified, export-controlled, proprietary, or employer-confidential information. Use only high-level, sanitized descriptions of prior work.
